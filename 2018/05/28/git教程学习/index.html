git教程学习

git的安装请自行百度，这里只进行git的基本操作介绍：

git首次安装的配置

首次使用git,需要进行如下配置：打开cmd命令行，输入：
git config –global user.name “用户名”
git config –global user.email “邮箱”

注意：这里的用户名和邮箱要用英文的
让它知道主人是谁，以后每次的提交都会用到这些信息

输入完成之后，在命令行中输入git config --list可以可以查看配置后的信息，若显示了name和email就说明配置成功了

git理论基础

git记录的是什么？
git记录的是每次版本的变动，原理则是将每个版本独立保存，它每个版本都是独立的存储，它看似消耗了很多空间，但是在分支管理上却带来了很多的益处和便利。
git实现版本控制的方法？

在git中有这样的三棵树：工作区、暂存区、仓库。
工作区就是我们通常使用的目录。
暂存区存放用户临时的改动，它本身是一个文件，保存的是即将提交到仓库的一个列表信息。
仓库是最终存放所有版本的位置，它有一个head指针，指向最新提交的内容。

git的工作流程

git的工作流程一般是酱紫的：

  1. 在工作目录中添加、修改文件
  2. 将需要进行版本管理的文件放入暂存区域
  3. 将暂存区域的文件提交到git仓库

git管理的文件有三种状态：
——已修改（modified）
——已暂存（staged）
——已提交（committed）

git实战

创建一个新的文件夹，打开cmd进入文件夹，

git init    //初始化仓库
