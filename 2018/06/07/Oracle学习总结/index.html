Oracle学习总结

这篇博客是Oracle课程中Oracle的启动和关闭、存储管理的总结，因为我觉得这部分是跟我之前学的数据库区别较大的一部分，也算是期末复习吧。
课程课本：数据库应用与设计 基于案例驱动的Oracle实现。
主要包括：

  • 数据库启动与关闭
  • 参数文件管理
  • 控制文件管理
  • 表空间和数据文件管理
  • 日志文件管理

Oracle的体系结构

Oracle是一个适合于中大型企业的数据库管理系统，其市场占有率是所有数据库管理系统中最高的，它的用户遍布银行、电信、航空、保险、金融等各个领域，基于此，公交公司管理信息系统决定采用Windows系统上的Oracle 10g数据库进行数据管理。

了解Oracle 10g的系统结构是怎样的？

Oracle的体系结构通常由数据库管理系统（DBMS）和数据库文件两部分组成，DBMS是由一组Oracle后台进程和一些服务器分配的内存空间组成，数据库文件则是一系列的物理文件。

查询test数据库的物理存储文件基本信息（至少包含文件名、扩展名和位置信息等）?

包括控制文件，日志文件和数据文件；扩展名：dbf；位置： D:\app\Administrator\oradata\test

Oracle数据库运行模式有哪两种，分别有什么含义？查询出test数据库是哪种运行模式？

  1. 归档方式(ARCHIVELOG):归档方式的目的在于当数据库发生故障时以最大限度恢复数据库,可以保证不丢失任何已提交的数据
  2. 不归档方式(NOARCHIVELOG):只能恢复数据库到最近的回收点(冷备份或是逻辑备份),数据丢失时非常可能的。在sys用户下输入archive log list;回车查。test数据库是不归档方式

test数据库最小存储单位是什么？大小？

test数据库最小存储单位Oracle数据块，大小由DB_BLOCK_SIZE初始化参数制定。

SGA是什么，由哪些部分组成？查询test数据库SGA当前值、最大值及各组成部分的大小？

SGA是系统全局区的英文简称，主要是用于存储数据库信息的内存区，该信息为数据库进程所共享（PGA不能共享的）。SGA主要由数据高速缓冲区（Database Buffer Cache）、共享池（Shared Pool）、重做日志缓冲区（Redo Log Buffer）、大型池（Large Pool）、Java池（Java Pool）、流池（Streams Pool）和其他结构（如固定SGA、锁管理等）组成。

什么是后台进程？Oracle的关键后台进程有哪些？如何产生多个DBWR进程来写数据？

后台进程负责保证数据库的稳定工作，每当数据库启动时，这些后台进程会自动启动，并且持续整 个实例的生命周期，每个进程负责一个独特的任务。PMON、SMON、DBWR、LGWR、ARCn、CKPT六大核心进程.无论哪个进程出现故障,都会导致数据库实例崩溃.如果杀掉某个进程,在ALERT LOG中会发现各种错误.等。DBWR进程启动的数目受到Oracle系统初始化参数DB_WRITER_PROCESSES的限制，默认值是1，可以通过修改这个变量保证更多的DBWR进程的启动来提高性能。

数据库启动与关闭

Oracle数据启动时要经历哪3个阶段?

（1） 启动实例
（2） 装载数据库
（3） 打开数据库

Oracle数据启动经历3个阶段分别做了什么工作？

（1）启动实例：启动实例时，读取初始化参数文件，为实例创建内存结构和后台进程，执行以下任务：
①读取初始化参数文件：如果启动命令指定PFILE子句，则根据该子句读取参数文件， 否则在系统默认位置读取服务器参数文件，如果找不到则读取文本参数文件。
②根据参数文件中的参数设置，分配SGA和PGA的内存空间。
③根据参数文件中的参数设置，启动后台进程。
在实例启动状态下，可以重建控制文件，重建数据库。
（2）装载数据库：装载数据库时，根据初始化参数文件中CONTROL_FILES参数的设置，找到控制文件，而且从控制文件中获取数据文件和重做日志文件的信息。如果控制文件找不到或损坏，装载数据库失败。
在装载数据库状态下，可以设置数据库的日志归档模式、执行数据库介质恢复、重新定位数据文件和重做日志文件。
（3）打开数据库：打开数据库时，根据控制文件中读取的信息打开所有联机的数据文件和重做日志文件，此时数据库才处于正常运行状态，用户能够与数据库建立连接并存取数据。如果某个数据文件或重做日志文件找不到或损坏，打开数据库失败。

Oracle数据库启动的命令有哪几种选项，它们的区别是什么？

STARUP [ NOMOUNT | MOUNT | OPEN | FORCE ]  [ RESTRICT ] [PFILE=file_name ]
  • NOMOUNT：只启动实例，不装载数据库，不打开数据库。
  • MOUNT：启动实例，装载数据库，不打开数据库。
  • OPEN：默认的启动选项，启动实例，装载数据库并打开数据库。
  • FORCE：用于数据库重启。无论数据库启动到哪一步或者启动失败，使用FORCE选项可以先强制关闭数据库实例，然后重新启动，并且不需要使用数据库关闭的命令。
  • RESTRICT：启动数据库后，只有RRESTRICTED SESSION权限的用户可以连接数据库。
  • PFILE：使用PFILE选项中指定的文本参数文件作为初始化参数文件启动数据库。

Oracle数据库关闭的命令有哪几种选项，它们的区别是什么？

SHUTDOWN [ NORMAL | IMMEDIATE | TRANSACTIONAL | ABORT ]
  • NORMAL：以正常方式关闭数据库。数据库在关闭前将检查所有的用户连接，并且不允许有新的用户连接，等待所有用户连接都断开后再关闭数据库。默认的关闭选项，SHUTDOWN语句没有指定任何选项即使用NORMAL选项。
  • IMMEDIATE：以立即方式关闭数据库。执行某些清除工作后才关闭数据库，不允许再有新的用户连接，所有用户未提交的事务回滚，终止回话，释放回话资源。当使用SHUTDOWN不能关闭数据库时，SHUTDOWN IMMEDIATE可以完成数据库的操作。
  • TRANSACTIONAL：等待事务终结后关闭数据库。介于NORMAL和IMMEDIATE两种方式之间。不允许再有新的用户连接，不允许产生新的事务，等待用户未提交的事务提交后断开连接。
  • ABORT：中止数据库。不允许再有新的用户连接，正在访问数据库的会话会被中止，所有用户未提交的事务不回滚。这是最快的关闭数据库方式，但是再次启动时要进行实例恢复，这些恢复操作同样是系统自动完成的，需要的时间较长。因此，应尽量避免使用ABORT选项关闭数据库。

参数文件管理

参数文件的作用是什么?

初始化参数文件是Oracle中一个很关键的文件，记录了数据库大量的配置参数，通过这些参数可以管理数据库和实例，进行数据库设计及性能调整。当Oracle数据库启动时，第一步就是从初始化参数文件中读取初始化参数以便启动实例。

Oracle 10g后有几种参数文件,它们之间有什么区别?

初始化参数文件有文本参数文件（PFILE）和服务器参数文件（SPFILE）两种形式。
主要的区别是：

  1. pfile是文本文件，spfile是二进制文件
  2. pfile可以通过文本编辑器修改，spfile主要是通过alter system set parameter=value [scope=memory|spfile|both]修改

Oracle 10g启动过程中在哪个阶段读取参数文件?

在数据库实例启动时读取。

Oracle 10g启动过程读取参数文件的顺序是什么?

oracle启动时，读取参数文件的顺序是：

1. spfile$ORACLE_SID.ORA(创建数据库时创建的spfile)
2. spfile.ora(oracle缺省spfile文件)
3. init$ORACLE_SID.ora(创建数据库时创建的pfile)
4. init.ora(oracle缺省pfile文件)

oracle会按顺序读出这些参数文件，如果在1没有找到就去找2，直到找到。

写出语句把当前Oracle的spfile文件内容转换为pfile文件,保存位置为C:\orclpfile.ora.

create pfile='c:\orclpfile.ora' from spfile='d:\app\Administrator\product\11.2.0\dbhome_1\database\SPFILEORCL.ora';

把C:\orclpfile.ora文件转换为当前spfile文件的默认位置替换.

create spfile from pfile='c:\orclpfile.ora';

用C:\orclpfile.ora文件进行startup启动数据库

startup pfile='c:\orclpfile.ora';

控制文件管理

控制文件的作用是什么?

控制文件是一个二进制文件。用于记录Oracle数据库的物理存储结构和当前状态。数据库的启动和正常运行都离不开控制文件。一旦控制文件损坏，数据库将不能正常运行。

Oracle数据库启动时什么阶段用到控制文件?

在数据库装载阶段，oracle会用到控制文件。

Oracle数据库中有效控制文件的个数和位置信息存放在什么地方?

每个控制文件只属于一个数据库，但为了防止数据丢失，一个数据库一般不止一个控制文件，实际的商用数据库一般有三个控制文件，最多可以有八个控制文件。控制文件放在“oracledate\实例名”目录下。

控制文件中包含哪些信息？这些信息由谁修改？控制文件破损数据库可以启动吗？

控制文件中包含了记录了数据库的物理存储结构和当前的状态，记录了数据文件和重做日志文件的名字及位置。
一个控制文件只属于一个数据库，创建数据库的同时创建控制文件。当更改数据库的物理存储结构时，Oracle会自动更新控制文件，任何用户包括数据库管理员都不能修改控制文件中的数据，控制文件由Oracle自动完成。

在磁盘上不同位置(2个以上不同的目录,其中一个目录为C:\<自己的学号>)创建多路控制文件，写出步骤和语句。

在C:\<自己的学号>目录下备份控制文件。

表空间和数据文件管理

数据库文件的作用是什么?表空间的作用是什么?

表空间是Oracle数据库中最大的逻辑存储结构，同时也是直接与数据库物理存储结构相关联的逻辑单位。
（1）它决定了数据库试题的空间分配
（2）设置数据库用户的空间份额
（3）控制数据库部分数据的可用性
（4）分布数据于不同的设备之间以改善性能
（5）备份和恢复数据

数据文件与表空间之间的关系是什么?

Oracle数据库的表空间在物理上表现为磁盘数据文件；也就是说Oracle数据库中的数据逻辑存储表空间中，物理存储在表空间包含的数据文件中。两者关系密切但又有些不同：
（1）一个表空间至少包含一个数据文件
（2）段可以跨多个数据文件，但是不能跨多个表空间。
（3）一个数据库必须有SYSTEM表空间和SYSTEM表空间，数据库创建时，系统自动把第一个数据文件分配给SYSTEM表空间。

Oralce 10g/11g安装后默认的表空间名称有哪些，其作用是什么？默认的数据文件位置是什么?

（1）SYSTEM表空间，系统创建数据库时自动创建，用于存储系统数据字典、系统管理信息、用户数据表、索引等对象。
（2）SYSAUX是10g新增的辅助SYSTEM表空间，由系统内部自动维护，不存储用户数据。
（3）UNDO表空间（撤销表空间）用来存储撤销信息，只能存回退段，不能存其他类型的段。
（4）USER表空间（用户表空间）主要存用户数据。
（5）TEMP临时表空间存储执行SQL语句时产生的临时数据（主要是排序或统计），一般此表空间所有用户通用。
默认的数据文件位置在 oradata\实例名 目录下

建立用户表空间tbsxxx(xxx为学号)，该表空间基于2个数据文件(%ORACLE_BASE\ oradata\orcl%\tbsxxx_01.dbf, %ORACLE_BASE\ oradata\orcl%\tbsxxx_02.dbf)，其大小均为50M，其中一个数据文件（02）存储空间可自动扩展5m，本地管理。设置tbsxxx为用户数据存储的默认表空间。验证创建结果。

表空间的状态有几种？在哪种状态下可以进行数据文件的移动？

（1）脱机，脱机下有三种模式：正常、临时、立即
（2）联机，
（3）只读
（4）读写
脱机状态可以对数据文件进行移动

日志文件管理

Oracle 重做日志文件的作用是什么?

数据库的日志文件通常被用来存放数据库变化信息。当数据被意外地删除或者修改时，或在出现实例故障或介质故障的情况下，可以利用日志文件进行恢复。

重做日志文件组与重做日志文件之间的关系是什么?

每个重做日志文件组由一组完全相同的重做日志文件组成，每个数据库要求至少有两个重做日志文件组，每组至少要包含一个重做日志文件。

Oracle数据库的日志归档模式有哪两种?默认的是那种?

数据库的日志归档模式有归档和非归档两种，默认的是不归档。
